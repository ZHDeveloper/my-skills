## 按场景推荐

### 1. 搜索、研究、信息获取

#### 日常搜索

- 优先用 `web-search-plus`
- 适合快速搜网页、新闻、资料、背景信息
- 自动路由到更合适的搜索能力，通常不需要自己纠结底层搜索源

#### 深度研究

- 优先用 `deep-research-pro`
- 适合需要长篇、结构化、带引用的研究结果
- 如果只是“帮我查一下”，通常没必要直接上它

#### 定向搜索

- `tavily` / `web-search-tavily`：更偏 AI 友好的结构化搜索
- 一般只有在你明确想用 Tavily，或者需要更可控的 Tavily 结果时再选
- 默认仍建议先用 `web-search-plus`

### 2. 热搜、新闻、趋势

#### 全网中文热搜

- 优先用 `hot-topics`
- 适合“今天有什么热点”“全网都在聊什么”这类问题
- 覆盖面最全，通常比单平台热搜更值得先看

#### 垂直资讯

- `ai-news-oracle`：AI 行业新闻
- `github-trending-cn`：GitHub 热门项目、开发者趋势
- `x-hot-topics-daily`：X 平台热点摘要，强调低成本获取重点信息
- `baidu-hot-cn`：只关心百度热搜时再用
- `daily-trending`：看通用热榜时可用，但通常被 `hot-topics` 覆盖

### 3. 股票、金融、投资

#### A 股 / 国内市场

| 场景 | 优先推荐 | 说明 |
| --- | --- | --- |
| 深度基本面分析 | `china-stock-analysis` | 更适合价值投资、财务解读、同行比较 |
| 行情与原始数据 | `akshare-stock` | 适合量化、历史数据、板块数据获取 |
| 实时价格查询 | `tencent-finance-stock-price` | 上手快、无需 API Key |
| 预警与盯盘 | `stock-monitor` | 适合规则化监控，如 RSI、均线、跳空等 |

#### 美股 / 全球市场

- 优先用 `us-stock-analysis`
- 如果只是查价格，用 `stock-price-checker` 更轻量
- 如果要综合看估值、基本面、技术面和对比，优先还是 `us-stock-analysis`

#### 加密货币 / 市场情绪

- `Crypto & Stock Market Data (Node.js)`：更偏市场数据获取
- `market-sentiment-pulse`：更偏情绪、舆情、市场氛围判断

### 4. 文档、报告、写作

#### 写报告、方案、PRD

- 优先用 `anygen-doc`
- 只要目标是“生成一份结构化文档内容”，一般都先想到它
- 适合：技术方案、调研报告、竞品分析、会议纪要、项目提案、白皮书

#### 编辑真实文件

- 优先用 `document-skills`
- 如果目标是操作文件本身，而不是只输出 Markdown 内容，就选它
- 细分场景：
  - `docx`：Word 文档
  - `xlsx`：表格与数据分析
  - `pdf`：PDF 提取、编辑、拆分、填写
  - `pptx`：演示文稿

#### 长文协作与写作流程

- `writing-assistant`：更像写作总控或写作团队协调器
- 适合大纲拆解、引用策略、长文协作
- 如果只是“直接给我一份文档”，优先仍是 `anygen-doc`

### 5. 思考、分析、决策

#### 通用复杂问题

- 优先用 `deep-thinking`
- 几乎所有“复杂、多步骤、有权衡”的问题都可以先选它

#### 特殊思考方式

- `first-principles-decomposer`：把问题拆到最底层假设，适合从头重构认知
- `mental-models`：从经济、心理、系统、数学等多学科同时看问题
- `Six Thinking Hats`：明确区分不同思考视角，适合团队讨论或方案比较
- `brainstorming`：偏创意发散、方案构思、前期探索
- `thinking-partner`：偏交互式共创，而不是直接给标准答案

### 6. 视频与多媒体

| 场景 | 优先推荐 | 说明 |
| --- | --- | --- |
| AI 视频生成 | `ai-video-gen` | 从提示词到视频的端到端方案 |
| 分镜脚本 | `ai-video-script` | 先产出短视频文案、镜头描述、提示词 |
| 代码生成视频 | `remotion-video-toolkit` | 用 React/Remotion 生成视频 |
| 命令式剪辑 | `FFmpeg Video Editor` | 想直接做裁剪、转码、压缩、抽音频 |
| 本地文本转视频流水线 | `video-generator` | 本地化自动视频生成 |

## 按类型整理

### 前端与 Web

#### 优先推荐

- `ui-design-system`：做 React 组件体系、Tailwind、Radix、shadcn/ui 时优先
- `nextjs-app-router-patterns`：做 Next.js App Router 时优先
- `vercel-react-best-practices`：偏 React/Next 性能与工程最佳实践

#### 补充技能

- `frontend-design`：强调视觉质量、审美完成度、非模板化表达
- `ui-ux-pro-max`：偏 UI/UX 规划、视觉方向、布局风格
- `tailwind-design-system`：偏 Tailwind 设计系统方法论
- `typescript-react-reviewer`：更适合 review 代码，而不是直接产出设计
- `webapp-testing`：验证本地 Web 应用行为

### iOS / Swift / Apple 生态

#### 代码与架构

- `swift-expert`：通用 Swift 开发能力
- `swift-concurrency`：遇到并发问题时优先级更高
- `ios-swift-development`：更偏 iOS 应用整体开发
- `swiftui-expert-skill`：SwiftUI 页面和组件开发首选
- `swiftui-performance-audit`：SwiftUI 性能问题首选

#### 文档与规范

- `axiom-apple-docs`：查 Apple API、编译错误、框架文档首选
- `axiom-apple-docs-research`：更偏研究 WWDC、全量资料和示例
- `ios-hig-design` / `mobile-ios-design`：做符合苹果规范的界面设计

#### 测试与模拟器

- `axiom-swift-testing`：Swift Testing 新测试框架
- `axiom-xctest-automation`：XCUITest 自动化
- `qa-testing-ios`：跑测、稳定测试、CI 测试控制
- `ios-simulator` / `ios-simulator-skill`：模拟器操作与自动化
- `ios-debugger-agent`：需要在模拟器里构建、运行、检查 App 行为时优先

### Python 与后端

- `django-expert`：Django / DRF 项目首选
- `fastapi-templates`：FastAPI 项目首选
- `async-python-patterns`：异步与并发问题
- `python-design-patterns`：架构设计与重构
- `python-performance-optimization`：性能瓶颈分析
- `pytest`：pytest 测试实践首选
- `python-testing-patterns`：更偏测试策略与模式
- `python-packaging`：项目打包、发版、CLI 发布

### 测试、调试、质量保障

- `systematic-debugging`：遇到 bug、异常、失败先用它梳理问题

- `github`：所有 GitHub CLI 类任务首选

## 相似技能优先级

这一节只讲“撞车时先选谁”。

### 搜索类

1. `web-search-plus`
2. `deep-research-pro`
3. `tavily` / `web-search-tavily`

选择原则：

- 普通搜索先 `web-search-plus`
- 需要研究报告再上 `deep-research-pro`
- 明确指定 Tavily 或需要 Tavily 风格结果时再用后两者

### 文档类

1. `anygen-doc`
2. `document-skills`
3. `writing-assistant`

选择原则：

- 生成内容优先 `anygen-doc`
- 操作真实文件优先 `document-skills`
- 长文流程管理或多阶段写作再用 `writing-assistant`

### 前端类

1. `ui-design-system`
2. `nextjs-app-router-patterns`
3. `frontend-design`
4. `vercel-react-best-practices`
5. `ui-ux-pro-max`

选择原则：

- 做组件体系先 `ui-design-system`
- 做 Next.js 架构先 `nextjs-app-router-patterns`
- 做视觉表现或高保真体验时补 `frontend-design`
- 做性能与模式修正时补 `vercel-react-best-practices`

### iOS / Swift 类

1. `swiftui-expert-skill`
2. `swift-expert`
3. `swift-concurrency`
4. `axiom-apple-docs`
5. `ios-debugger-agent`

选择原则：

- SwiftUI 页面优先 `swiftui-expert-skill`
- 语言和架构问题先 `swift-expert`
- 并发隔离、actor、Sendable 问题优先 `swift-concurrency`
- 查文档和 API 用 `axiom-apple-docs`
- 真机/模拟器运行调试用 `ios-debugger-agent`

### Python / 后端类

1. `django-expert` 或 `fastapi-templates`
2. `pytest`
3. `async-python-patterns`
4. `python-design-patterns`
5. `python-performance-optimization`

选择原则：

- 先按框架选主 skill
- 测试用 `pytest`
- 异步问题补 `async-python-patterns`
- 性能瓶颈补 `python-performance-optimization`

### 自动化 / Agent 编排类

1. `dispatching-parallel-agents`
2. `agent-team-orchestration`
3. `n8n-workflow-automation`
4. `automation-workflows`
5. `workflow-orchestration-patterns`

选择原则：

- 独立任务并行优先 `dispatching-parallel-agents`
- 需要团队角色分工再上 `agent-team-orchestration`
- 需要落地自动化时，工程化选 `n8n-workflow-automation`，业务自动化选 `automation-workflows`

## 如何触发与使用

### 最简单的用法

直接描述需求即可，例如：

- “帮我深度研究一下特斯拉最近财报，给我带引用的结论。”
- “请用 `ui-design-system` 帮我做一个设置页面组件。”
- “用 `hot-topics` 看看今天中文互联网都在聊什么。”
- “用 `anygen-doc` 生成一份竞品分析文档。”

### 推荐写法

可以按这个模板表达：

```text
请使用 <skill 名称>，帮我完成 <任务目标>。
如果有多个候选技能，优先选最合适的那个，并说明原因。
```

示例：

```text
请使用 deep-research-pro，帮我研究 OpenAI、Anthropic、Google 三家的模型策略差异，并给出引用来源。
```

```text
请使用 china-stock-analysis，分析一下宁德时代和比亚迪的基本面差异。
```

```text
请使用 document-skills，把这份 PDF 里的表格提取出来整理成 Excel。
```

### 不确定用哪个时

如果你自己也不确定，可以直接说：

- “这件事该用哪个 skill？”
- “先帮我选 skill，再执行。”
- “如果有相似能力，请优先用最稳妥的那个。”

通常这种情况优先考虑：

- `find-skills`：先发现能力
- 再根据任务性质选最合适的主 skill

- 搜索先 `web-search-plus`
- 研究先 `deep-research-pro`
- 写文档先 `anygen-doc`
- 改文件先 `document-skills`
- 复杂问题先 `deep-thinking`

- Next.js 先 `nextjs-app-router-patterns`
- SwiftUI 先 `swiftui-expert-skill`
- 股票分析按市场选 `china-stock-analysis` / `us-stock-analysis`
- 不知道有没有现成能力，先 `find-skills`
